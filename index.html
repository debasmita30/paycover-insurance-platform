Database Schema (PostgreSQL)
-- Users Table CREATE TABLE users ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), name VARCHAR(100) NOT NULL, email VARCHAR(150) UNIQUE NOT NULL, age INT CHECK (age >= 18 AND age <= 80), income_band VARCHAR(20), city_tier VARCHAR(10), family_size VARCHAR(20), ab_variant CHAR(1) DEFAULT 'A', created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); -- Insurance Plans Table CREATE TABLE insurance_plans ( id VARCHAR(10) PRIMARY KEY, insurer VARCHAR(100), type VARCHAR(30), name VARCHAR(150), monthly_premium DECIMAL(10,2), annual_premium DECIMAL(10,2), sum_insured BIGINT, csr DECIMAL(5,2), -- Claim Settlement Ratio network_hospitals INT, rating DECIMAL(3,1), cashless BOOLEAN, maternity BOOLEAN, digital BOOLEAN, telemedicine BOOLEAN, created_at TIMESTAMPTZ DEFAULT NOW() ); -- User Sessions (Analytics) CREATE TABLE sessions ( session_id VARCHAR(20) PRIMARY KEY, user_id UUID REFERENCES users(id), ab_variant CHAR(1), device VARCHAR(20), started_at TIMESTAMPTZ DEFAULT NOW(), last_active TIMESTAMPTZ DEFAULT NOW(), events_count INT DEFAULT 0 ); -- Events (Mixpanel-style) CREATE TABLE events ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), session_id VARCHAR(20), event_name VARCHAR(100), properties JSONB, ab_variant CHAR(1), timestamp TIMESTAMPTZ DEFAULT NOW() ); -- Quotes CREATE TABLE quotes ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), user_id UUID REFERENCES users(id), plan_id VARCHAR(10) REFERENCES insurance_plans(id), premium DECIMAL(10,2), status VARCHAR(20) DEFAULT 'pending', created_at TIMESTAMPTZ DEFAULT NOW() );

🎯 Executive Summary
PayCover is PIBPL's intelligent insurance comparison and recommendation product targeting India's 500M+ underinsured population. The platform aggregates 50+ plans across 4 insurance categories and uses a proprietary AI scoring engine to surface the best-fit plan for each user's unique risk profile — replacing the industry's existing "browse and guess" model with a "profile and recommend" model.

🔍 Problem Statement
Core Problem: India has one of the world's lowest insurance penetration rates (~4.2% of GDP vs 11% global average). The primary barriers are not affordability — they are awareness, trust, and decision-making complexity.

Users on Paytm Insurance today face: 50+ plans with similar-sounding names, dense comparison tables, heavy agent dependency, and no personalised guidance. The result: 87% of users who enter the funnel never purchase.

⚙️ Technical Architecture
Frontend: React 18 + TypeScript + Tailwind CSS. Deployed on Cloudflare Edge (CDN).
Backend: Node.js + Express REST API. Containerised with Docker, orchestrated on Kubernetes (GKE).
Database: PostgreSQL (primary) + Redis (cache/sessions) + BigQuery (analytics).
AI Engine: Python FastAPI service. Rule-based scoring (v1) → ML model (v2 roadmap).
Payments: Paytm Payment Gateway (direct integration, no redirect).
Analytics: Mixpanel (product events) + Google Analytics 4 (marketing attribution).

🔐 Compliance & Security
IRDAI Compliance: All plans must display IRDAI registration number. Mandatory disclosure before purchase (as per Insurance Act 1938).

Data Security: All PII encrypted at rest (AES-256) and in transit (TLS 1.3). DPDP Act 2023 compliant — explicit consent, data minimisation, right to erasure.

KYC/AML: Aadhaar-based eKYC (UIDAI API) for policies above ₹50,000 annual premium.
